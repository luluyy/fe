# React 虚拟 DOM 原理

## 什么是虚拟 DOM？

虚拟 DOM（Virtual DOM）是对真实 DOM 结构的 JavaScript 对象描述（通常称 VNode / React Element 树）。

React 每次更新时会先生成一棵新的虚拟树，再和旧树比较，最后把最小变更提交到真实 DOM。

## 为什么不用每次直接操作真实 DOM？

- 真实 DOM 操作成本高（布局、重绘、回流相关）。
- 先在 JS 层做比较与批量计算，能更好控制更新粒度。
- 配合 Fiber 调度，可中断、可恢复，提升大应用交互流畅度。

> 注意：虚拟 DOM 不是“永远更快”，而是让复杂 UI 更新更可预测、更可调度。

## 工作流程（简化）

1. state/props 变化触发 render
2. 生成新的虚拟 DOM 树
3. 与旧树做 diff（Reconciliation）
4. 得到最小变更集合
5. commit 阶段更新真实 DOM

## 面试常见追问

### Q1：虚拟 DOM 的优势是“快”吗？

不是单纯“快”，核心价值是：**跨平台抽象 + 可控更新 + 调度能力**。

### Q2：有了虚拟 DOM 为什么还要 key？

因为列表 diff 需要稳定身份标识，不然容易错位复用。

## 30 秒口述模板

虚拟 DOM 是真实 DOM 的 JS 抽象。React 更新时先生成新虚拟树，再和旧树 diff，最后只把必要变更提交到真实 DOM。它的核心价值不只是性能，更是让更新过程可预测、可调度，并支撑复杂应用的工程化。
