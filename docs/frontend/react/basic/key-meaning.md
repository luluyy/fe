# 为什么 React 列表需要 key？

`key` 用于在 diff 时标识元素身份，帮助 React 复用节点并减少错误更新。

## 为什么不用 key 会出问题？

如果 key 不稳定，React 可能把“旧节点状态”错误复用到“新位置节点”上，出现输入框错位、选中态错乱等问题。

## 代码示例

```tsx
{list.map((item) => (
  <Row key={item.id} item={item} />
))}
```

推荐用业务唯一 id 作为 key。

## index 当 key 什么时候可以用？

可以用在**静态列表**且**不会插入/删除/排序**的场景。  
只要列表顺序可能变化，就尽量不要用 index。

## 面试回答要点

1. key 作用：标识元素身份，不是给组件读的 props。
2. key 原则：稳定、唯一、可预测。
3. 错误示例：随机 key 会导致每次都卸载重建。

## 20 秒口述模板

key 是 React diff 时识别列表元素身份的依据。稳定 key 能正确复用节点和状态；不稳定 key（如随机数、变化中的 index）会导致重建和状态错位。优先使用业务 id。
