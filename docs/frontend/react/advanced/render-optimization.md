# React 渲染优化思路

先记住一句话：**先定位瓶颈，再做优化；不要“全量 memo 化”。**

## 常见优化手段

1. 组件拆分，缩小更新影响面。
2. `React.memo` 减少无效子组件重渲染。
3. `useMemo` 缓存重计算结果。
4. `useCallback` 稳定函数引用（常配合 `memo`）。
5. 列表虚拟化（如 react-window）处理长列表。
6. 避免不必要状态提升，状态尽量就近。

## 典型场景

- 父组件频繁更新导致子组件反复渲染。
- 复杂计算每次 render 都执行。
- 大列表滚动卡顿。

## 高频追问

### Q1：`memo` 为什么有时没效果？

如果传入子组件的 props 每次都是新引用（如新对象/新函数），浅比较会失败，`memo` 仍会重渲染。

### Q2：是不是所有组件都应该 `memo`？

不是。`memo` 也有比较成本，小组件或低频更新组件可能得不偿失。

### Q3：优化顺序建议？

先用 Profiler 找热点，再做组件拆分、状态下沉、最后上 memo 系列。

## 30 秒口述模板

React 渲染优化核心是减少无效渲染和重计算。实践上先做组件拆分与状态就近管理，再用 `memo/useMemo/useCallback` 精准优化，长列表用虚拟化。优化前先定位瓶颈，避免过度优化。
