TypeScript 进阶 20 题（详细版）

1) 条件类型​

代码示例​

type IsString<T> = T extends string ? true : false;

原理讲解​

按类型关系进行分支推导，是高级类型基础。

练习​

  • 基础练习：写一个 IsArray<T>。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

什么时候会触发分布式？​

当 T 是联合且裸类型参数出现在条件左侧时。

面试口述模板​

条件类型是类型计算的 if/else。

2) infer 提取类型​

代码示例​

type RT<T> = T extends (...args:any[]) => infer R ? R : never;

原理讲解​

在条件类型里提取局部类型变量。

练习​

  • 基础练习：写 MyParameters<T>。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

可以 infer 多次吗？​

可以，在不同位置提取参数和返回等信息。

面试口述模板​

infer 是做类型“解构”的关键字。

3) 分布式条件类型​

代码示例​

type ToArray<T> = T extends any ? T[] : never;

原理讲解​

联合成员会逐个分发后再合并。

练习​

  • 基础练习：验证 string|number 的结果。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

如何关闭分发？​

用元组包裹：[T] extends [U]。

面试口述模板​

理解分发能避免很多类型误判。

4) 映射类型修饰符​

代码示例​

type Mutable<T> = { -readonly [K in keyof T]-?: T[K] };

原理讲解​

可批量增删 readonly 和 optional。

练习​

  • 基础练习：实现 DeepPartial<T>。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

应用场景？​

DTO 转换、表单编辑态类型建模。

面试口述模板​

映射修饰符是批量改造类型结构的利器。

5) 模板字面量类型​

代码示例​

type EventName<K extends string> = `on${Capitalize<K>}`;

原理讲解​

在类型层拼接字符串，约束命名规则。

练习​

  • 基础练习：实现 ApiPath<'user'|'post'>。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

实际场景？​

事件名、路由 key、权限点、CSS 命名约束。

面试口述模板​

模板字面量让字符串协议类型安全。

6) 联合过滤工具​

代码示例​

type A = Exclude<'a'|'b','a'>;

原理讲解​

Exclude/Extract/NonNullable 本质都是条件过滤。

练习​

  • 基础练习：写 OnlyString<T>。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

Extract 何时用？​

从联合中抽取满足条件子集。

面试口述模板​

先会原理再用工具，理解更稳。

7) 手写 ReturnType/Parameters​

代码示例​

type MyParameters<T> = T extends (...args: infer P) => any ? P : never;

原理讲解​

通过 infer 从函数签名提取信息。

练习​

  • 基础练习：手写 ConstructorParameters。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

构造函数版本怎么写？​

T extends new (...args: infer P) => any ? P : never。

面试口述模板​

函数签名提取题是二面高频。

8) 协变与逆变​

代码示例​

type Fn<T> = (x:T)=>void;

原理讲解​

参数与返回值位置兼容方向不同。

练习​

  • 基础练习：比较 (x:Animal)=>void 与 (x:Dog)=>void 可赋值性。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

strictFunctionTypes 影响？​

函数参数兼容检查更严格。

面试口述模板​

这题常用于考察类型系统深度。

9) any 污染治理​

代码示例​

let x:any = 1; x.foo.bar();

原理讲解​

any 会吞掉下游类型，风险传播快。

练习​

  • 基础练习：找出代码中 any 传播链并替换。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

治理步骤？​

禁新增 any -> 边界 unknown -> 渐进收敛。

面试口述模板​

治理 any 是 TS 成熟度关键指标。

10) unknown 边界收敛​

代码示例​

function parse(v: unknown){ if(typeof v==='string') return v.length; }

原理讲解​

unknown 强制先校验再使用。

练习​

  • 基础练习：为 API 响应写守卫函数。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

如何配 runtime 校验？​

zod/io-ts 先 parse 再 infer 类型。

面试口述模板​

边界用 unknown 比 any 安全得多。

11) 声明合并​

代码示例​

interface Window { appVersion: string }

原理讲解​

同名 interface 会合并，用于扩展类型。

练习​

  • 基础练习：给全局 Window 扩展一个字段。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

如何防污染？​

限制声明文件作用域并最小化扩展。

面试口述模板​

声明合并常见于全局增强场景。

12) 模块增强​

代码示例​

declare module 'axios' { interface AxiosRequestConfig { traceId?: string } }

原理讲解​

为第三方模块补充额外字段定义。

练习​

  • 基础练习：给常用库补一个自定义字段类型。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

不生效怎么排查？​

检查 module 名称、d.ts include 和声明可见性。

面试口述模板​

模块增强是企业项目常见需求。

13) d.ts 与 declare​

代码示例​

declare const VERSION: string;

原理讲解​

只声明类型，不生成运行时代码。

练习​

  • 基础练习：给无类型库写最小声明。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

何时写 ambient declaration？​

无类型包、全局注入对象时。

面试口述模板​

声明文件是 TS 接第三方生态的桥梁。

14) 类型驱动开发​

代码示例​

type UserDTO = {id:string;name:string};

原理讲解​

先定义契约再实现逻辑，降低重构风险。

练习​

  • 基础练习：以类型先行设计一个接口层。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

如何与后端同步？​

OpenAPI/GraphQL codegen 自动生成。

面试口述模板​

类型先行能减少沟通歧义。

15) 类型安全事件总线​

代码示例​

type E = { login:{id:string} };

原理讲解​

用映射类型绑定事件名和 payload。

练习​

  • 基础练习：实现 on/emit 泛型签名。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

关键签名是什么？​

on<K extends keyof E>(k:K,h:(p:E[K])=>void)。

面试口述模板​

这题可体现泛型设计能力。

16) API 返回值收敛​

代码示例​

// zod.safeParse(data)

原理讲解​

类型系统 + 运行时校验双保险。

练习​

  • 基础练习：给接口响应写 schema 校验。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

为什么不能只靠 as？​

as 不做运行时校验，脏数据会延迟爆炸。

面试口述模板​

边界安全决定线上稳定性。

17) 编译性能优化​

代码示例​

// incremental / skipLibCheck / references

原理讲解​

通过增量构建和拆分工程降低编译时长。

练习​

  • 基础练习：为 monorepo 制定编译优化方案。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

project references 怎么做？​

按包拆 tsconfig，开启 composite/incremental。

面试口述模板​

性能优化是大型 TS 项目刚需。

18) 类型体操风险​

代码示例​

// 深递归条件类型可能拖慢编译

原理讲解​

类型复杂度过高会损害可读性与构建性能。

练习​

  • 基础练习：把一段复杂类型改写成可维护版本。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

何时应降级复杂度？​

当理解成本或编译成本明显过高时。

面试口述模板​

类型要服务业务，不是炫技。

19) strict 治理策略​

代码示例​

// CI: tsc --noEmit

原理讲解​

通过规则分阶段收紧，稳步提升质量。

练习​

  • 基础练习：设计一个 strict 灰度计划。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

老项目怎么推？​

先告警后阻断，按目录逐批收口。

面试口述模板​

治理要可执行、可度量。

20) 迁移与收益评估​

代码示例​

// baseline -> module -> full

原理讲解​

TS 落地是持续工程化过程。

练习​

  • 基础练习：定义迁移 KPI（错误率/交付效率）。
  • 进阶练习：补一个边界场景（空值/并发/错误处理）并解释。

高频追问​

如何量化收益？​

看故障率、重构效率、PR 质量和回归成本。

面试口述模板​

迁移成功标准是稳定性与效率双提升。
