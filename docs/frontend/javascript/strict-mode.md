# 什么是严格模式和非严格模式？

- 非严格模式：JavaScript 的宽松模式，会容忍一些不规范写法。
- 严格模式：更严格的执行模式，很多潜在问题会直接抛错。

## 如何开启严格模式

```js
'use strict';
```

另外，ES Module（`import/export`）默认就是严格模式。

## 高频差异（面试常考）

### 1) 未声明变量赋值

```js
function foo() {
  x = 1; // 未声明
}
```

- 非严格：可能变成全局变量
- 严格：直接报错

### 2) 普通函数中的 this

```js
function foo() {
  console.log(this);
}
foo();
```

- 非严格：浏览器里通常是 `window`
- 严格：`undefined`

### 3) 删除不可删除属性

- 非严格：删除失败但通常不抛错
- 严格：抛错

### 4) 函数参数重名

- 非严格：部分场景可通过
- 严格：报错

## 为什么工程里更推荐严格模式

- 更早暴露 bug
- 避免隐式全局变量污染
- 行为更可预测
- 更利于团队协作与长期维护

## 20 秒口述模板

严格模式是 JS 的安全执行模式，通过 `'use strict'` 或模块默认启用。它会禁止隐式全局、收紧 this 行为并让不规范写法报错，从而减少线上隐患。
