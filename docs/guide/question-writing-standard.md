# 技术要点编写规范

后续所有题目建议统一按这套结构，保证可读、可背、可追问。

## 1. 必备三件套

1. **代码示例**：最小可运行，尽量 10-30 行。
2. **原理讲解**：说明机制，不只给结论。
3. **追问列表**：至少 3 条，且每条必须附“参考答案”。

## 2. 推荐结构

- 题目
- 标准回答（1-2 分钟）
- 原理讲解
- 代码示例
- 代码拆解
- 深挖追问
- 深挖追问对应答案
- 常见误区
- 技术评审者继续追问
- 复盘

## 3. 代码示例要求

- 优先 TS/TSX（与项目栈一致）。
- 变量命名清晰，避免无意义缩写。
- 关键行要有注释。
- 给出“期望输出/结果”。

## 4. 追问设计建议

- 机制追问：为什么这么设计？
- 边界追问：极端输入会怎样？
- 工程追问：线上如何落地、怎么监控/排查？
- 每条追问后必须紧跟 1-3 句参考答案，避免“只有问题没有可背答案”。

## 5. 一句话原则

每道题都要做到：**看得懂、讲得出、写得出、追得下去**。
