Vite vs Webpack

一屏看懂：启动、HMR、生态、定制、迁移成本（用于面试速记）

开发时流程图

Vite 浏览器请求模块 -> Dev Server 按需转换 -> 返回 ESM -> 页面更新

Webpack Entry 出发 -> 构建依赖图 -> 内存打包 -> 返回 bundle -> 页面更新

关键词：Vite = bundleless dev；Webpack = bundled dev

面试口述模板（40 秒）

Vite 在开发阶段更快，本质是原生 ESM + 按需转换，避免先全量打包； Webpack 的优势在复杂企业场景的生态与可定制能力。我的选型标准是： 新项目优先 Vite，历史包袱重且强依赖插件链时优先 Webpack， 最后用启动时间、热更新时间、构建时长和线上回归数据做量化决策。

现场可加：bundleless != 不打包生产

选择决策图

Q1: 新项目吗？ 是 -> Vite 倾向更高
Q2: 强依赖 Webpack 插件链 / 模块联邦？ 是 -> Webpack 稳妥；否 -> 继续看 Vite
Q3: 目标是提升本地开发体验？ 是 -> 优先 Vite
